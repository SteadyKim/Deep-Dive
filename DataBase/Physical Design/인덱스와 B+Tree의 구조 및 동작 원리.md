# 인덱스와 B+Tree의 구조 및 동작 원리

## 개요

인덱스는 테이블의 모든 데이터를 순차적으로 읽지 않고도 원하는 데이터를 빠르게 찾기 위해 사용하는 물리적 저장 구조입니다.  
데이터베이스는 인덱스를 통해 조건에 맞는 데이터의 위치를 먼저 찾고, 이후 필요한 테이블 블록만 접근함으로써 디스크 입출력을 줄입니다.

## 인덱스 개념

인덱스는 일반적으로 다음 정보를 기준으로 관리됩니다.

- 인덱스 키 값
- 해당 데이터가 저장된 위치 정보인 `ROWID`

즉, 특정 컬럼 값을 기준으로 정렬된 탐색 구조를 만들어 두고, 실제 데이터가 저장된 위치를 빠르게 찾도록 돕는 역할을 합니다.  
쿼리 수행 시 데이터베이스는 테이블 전체를 읽는 대신 인덱스를 우선 탐색하여 필요한 데이터만 선택적으로 접근합니다.

## B+Tree 구조

B+Tree는 오라클을 포함한 관계형 데이터베이스에서 가장 널리 사용하는 인덱스 구조입니다.  
기본적으로 `루트 노드`, `브랜치 노드`, `리프 노드`로 구성됩니다.

### 루트 노드와 브랜치 노드

루트와 브랜치 노드는 하위 노드로 이동하기 위한 탐색 정보만 저장합니다.

- 키 값의 범위
- 하위 노드의 블록 주소

이 노드들은 실제 데이터의 위치인 `ROWID`를 저장하지 않습니다.  
그 결과 하나의 블록에 더 많은 분기 정보를 담을 수 있고, 트리의 높이를 낮게 유지할 수 있습니다.

### 리프 노드

리프 노드에는 실제 조회에 필요한 정보가 저장됩니다.

- 인덱스 키 값
- 테이블 블록의 물리적 주소인 `ROWID`

모든 리프 노드는 양방향 연결 리스트 형태로 연결됩니다.  
이 구조 덕분에 특정 시작 지점을 찾은 뒤, 인접한 리프 노드로 이동하며 연속된 데이터를 효율적으로 읽을 수 있습니다.

## B+Tree의 특징

- 모든 리프 노드가 동일한 깊이에 위치하는 밸런스드 트리 구조입니다.
- 데이터가 증가해도 탐색 경로 길이가 크게 흔들리지 않아 안정적인 성능을 제공합니다.
- 탐색 성능은 일반적으로 `O(log n)`입니다.
- 리프 노드 간 연결 구조 덕분에 범위 검색에 매우 유리합니다.

## B+Tree 탐색 원리

B+Tree 탐색은 크게 `수직적 탐색`과 `수평적 탐색`으로 나눌 수 있습니다.

### 수직적 탐색

수직적 탐색은 루트 블록에서 시작하여 브랜치 블록을 거쳐 리프 블록까지 내려가는 과정입니다.

- 찾고자 하는 키 값이 어느 범위에 속하는지 판단합니다.
- 해당 범위를 포함하는 하위 블록으로 이동합니다.
- 최종적으로 조건에 부합하는 데이터의 시작 리프 블록을 찾습니다.

이 단계는 조건을 만족하는 데이터가 어디서부터 시작되는지를 결정하는 과정입니다.

### 수평적 탐색

수평적 탐색은 수직적 탐색으로 찾아낸 리프 블록의 시작점부터 조건이 끝나는 지점까지 옆으로 이동하며 읽는 과정입니다.

- 인접한 리프 노드를 연결 리스트로 따라갑니다.
- 범위 조건에 해당하는 인덱스 엔트리를 연속적으로 읽습니다.

예를 들어 `BETWEEN`, `>`, `<`, `LIKE 'ABC%'`와 같은 범위 조건에서 매우 효율적으로 동작합니다.

## B-Tree와 B+Tree의 차이점

### 데이터 저장 위치

- `B-Tree`: 루트나 브랜치 같은 내부 노드에도 실제 데이터 또는 `ROWID`를 저장할 수 있습니다.
- `B+Tree`: 실제 데이터 위치 정보는 오직 리프 노드에만 저장합니다.

### 탐색 경로

- `B-Tree`: 중간 노드에서 원하는 값을 찾으면 탐색이 종료될 수 있습니다.
- `B+Tree`: 모든 탐색이 반드시 리프 노드까지 내려가므로 탐색 경로가 일정합니다.

즉, B+Tree는 조회 응답 시간이 상대적으로 더 균일합니다.

### 범위 스캔 효율

- `B-Tree`: 리프 노드 간 순차 연결이 일반적이지 않아 범위 검색 시 비효율이 생길 수 있습니다.
- `B+Tree`: 리프 노드가 연결되어 있어 조건 범위를 따라 연속적으로 읽기 쉽습니다.

따라서 B+Tree는 단건 조회뿐 아니라 범위 검색에서도 매우 강한 구조입니다.

## 정리

- 인덱스는 빠른 조회를 위해 키 값과 데이터 위치 정보를 관리하는 물리적 구조입니다.
- B+Tree는 루트, 브랜치, 리프 노드로 구성되며 실제 데이터 위치 정보는 리프에 저장합니다.
- 탐색은 리프까지 내려가는 수직적 탐색과 리프를 따라 이동하는 수평적 탐색으로 이루어집니다.
- 리프 노드 연결 구조 덕분에 B+Tree는 범위 검색에 특히 적합합니다.
- 관계형 데이터베이스의 일반적인 인덱스 구조를 이해할 때 B+Tree는 가장 핵심적인 개념입니다.
