# SGA와 PGA

## 개요

Oracle 데이터베이스의 메모리 구조를 이해할 때 가장 먼저 구분해야 하는 개념이 `SGA`와 `PGA`입니다.

- `SGA(System Global Area)`는 여러 세션과 백그라운드 프로세스가 함께 사용하는 공유 메모리 영역입니다.
- `PGA(Program Global Area)`는 개별 서버 프로세스가 독립적으로 사용하는 전용 메모리 영역입니다.

두 메모리는 모두 성능에 직접적인 영향을 주지만, **누가 사용하는지**, **무엇을 저장하는지**, **언제 할당되고 해제되는지**가 다릅니다.

## 한눈에 비교

| 구분 | SGA | PGA |
| --- | --- | --- |
| 성격 | 공유 메모리 | 프로세스 전용 메모리 |
| 할당 시점 | 오라클 인스턴스 시작 시 | 서버 프로세스 생성 시 |
| 사용 주체 | 모든 세션과 백그라운드 프로세스 | 특정 서버 프로세스 |
| 공유 여부 | 공유됨 | 공유되지 않음 |
| 해제 시점 | 인스턴스 종료 시 | 프로세스 종료 시 |
| 주요 역할 | 데이터 캐싱, SQL 공유, 트랜잭션 로그 버퍼링 | 정렬, 해시 연산, 세션 작업 메모리 |

## SGA란 무엇인가

`SGA`는 오라클 인스턴스가 시작될 때 물리 메모리에 할당되는 공유 영역입니다.  
데이터베이스에 접속하는 모든 세션과 백그라운드 프로세스는 이 영역에 공동으로 접근하며, 필요한 데이터를 함께 사용합니다.

즉, SGA는 데이터베이스 전체가 공통으로 참조하는 작업 공간에 가깝습니다.

### SGA의 대표 구성 요소

#### 1. Buffer Cache

디스크에서 읽어온 데이터 블록을 메모리에 캐싱하는 영역입니다.  
같은 블록에 다시 접근할 때 디스크 I/O를 줄여 성능을 높입니다.

#### 2. Shared Pool

파싱된 SQL 문장과 실행 계획을 보관하는 영역입니다.  
이미 실행한 SQL의 파싱 결과를 재사용할 수 있어 불필요한 파싱 비용을 줄일 수 있습니다.

#### 3. Redo Log Buffer

트랜잭션 변경 사항을 기록하는 메모리 버퍼입니다.  
이 내용은 이후 `LGWR` 등에 의해 리두 로그 파일로 기록되어 장애 복구의 기반이 됩니다.

## PGA란 무엇인가

`PGA`는 클라이언트 요청에 따라 개별 서버 프로세스가 생성될 때 할당되는 독립 메모리 영역입니다.  
특정 프로세스 전용으로 사용되며, 다른 프로세스와 공유되지 않습니다.

해당 프로세스가 종료되면 운영체제로 메모리가 반환됩니다.

즉, PGA는 "공유 메모리"가 아니라, **각 서버 프로세스가 자신의 작업을 수행하기 위해 따로 들고 있는 작업 공간**입니다.

### PGA의 대표 구성 요소

#### 1. Sort Area

`ORDER BY`, `GROUP BY` 같은 연산을 수행할 때 정렬 작업에 사용되는 영역입니다.

#### 2. Hash Area

해시 조인이나 해시 기반 연산을 수행할 때 사용하는 메모리 영역입니다.

#### 3. 세션 메모리

커서 상태, 바인드 변수 값 등 세션 실행에 필요한 개별 정보를 저장합니다.

## 쿼리 실행 시 SGA와 PGA의 상호작용

사용자가 쿼리를 요청하면 서버 프로세스는 먼저 `SGA`를 활용해 공통 자원을 확인합니다.

### 1. Shared Pool 확인

서버 프로세스는 `Shared Pool`에서 동일한 SQL이 이미 파싱되어 있는지 확인합니다.  
기존 실행 계획이 있다면 이를 재사용하고, 없다면 새로 파싱합니다.

### 2. Buffer Cache 접근

이후 필요한 데이터를 읽기 위해 `Buffer Cache`를 확인합니다.  
필요한 블록이 이미 캐시에 있으면 메모리에서 바로 읽고, 없으면 디스크에서 읽어와 캐시에 적재합니다.

### 3. PGA에서 작업 수행

조회 결과에 `ORDER BY`, `GROUP BY`, 해시 조인 같은 추가 작업이 포함되어 있다면, 서버 프로세스는 필요한 데이터를 자신의 `PGA`로 가져와 연산을 수행합니다.

이 과정에서 다음과 같은 작업이 발생할 수 있습니다.

- `Sort Area`에서 정렬 수행
- `Hash Area`에서 해시 조인 수행
- 세션 메모리에서 커서 상태와 바인드 변수 관리

### 4. 결과 반환

연산이 끝나면 서버 프로세스는 최종 결과를 클라이언트에 반환합니다.

## 흐름으로 보면

```text
사용자 쿼리 요청
→ Shared Pool에서 SQL/실행 계획 확인
→ Buffer Cache에서 데이터 블록 확인
→ 필요 시 디스크에서 읽어 캐시에 적재
→ ORDER BY / GROUP BY / HASH JOIN 작업은 PGA에서 수행
→ 결과 반환
```

## 정리

- `SGA`는 여러 세션이 함께 사용하는 공유 메모리입니다.
- `PGA`는 각 서버 프로세스가 독립적으로 사용하는 전용 메모리입니다.
- `SGA`는 데이터 캐싱과 SQL 재사용에 강점이 있습니다.
- `PGA`는 정렬, 해시 조인, 세션별 작업 처리에 사용됩니다.

결국 오라클은 `SGA`를 통해 공통 자원을 효율적으로 공유하고, `PGA`를 통해 각 프로세스의 개별 작업을 처리하면서 전체 성능을 최적화합니다.
