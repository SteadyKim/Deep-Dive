# Buffer Cache와 LRU 알고리즘

## 개요

데이터베이스는 디스크 I/O에서 가장 큰 지연이 발생합니다.  
이를 줄이기 위해 자주 접근하는 데이터 블록을 메모리 영역인 `Buffer Cache`에 보관하고, 가능한 한 디스크 대신 메모리에서 먼저 데이터를 읽도록 동작합니다.

## Buffer Cache의 목적

`Buffer Cache`의 핵심 목적은 **디스크 접근을 최소화하여 응답 속도를 높이는 것**입니다.

사용자가 데이터를 요청하면 시스템은 먼저 `Buffer Cache`를 확인합니다.

- `Cache Hit`: 요청한 데이터 블록이 이미 메모리에 존재하는 상태
- `Cache Miss`: 요청한 데이터 블록이 메모리에 없어 디스크에서 읽어와야 하는 상태

`Cache Hit`가 발생하면 메모리에서 즉시 데이터를 읽을 수 있으므로 응답 속도가 빠릅니다.  
반대로 `Cache Miss`가 발생하면 물리적 디스크에서 블록을 읽어 `Buffer Cache`에 적재한 뒤 사용자에게 전달해야 하므로 더 많은 시간이 소요됩니다.

즉, `Buffer Cache`는 데이터베이스 성능을 높이기 위한 가장 기본적이고 중요한 메모리 최적화 장치라고 볼 수 있습니다.

## LRU 알고리즘의 원리

`Buffer Cache`는 메모리 공간이 무한하지 않기 때문에, 새로운 데이터를 적재할 공간이 부족해지면 기존 블록 중 일부를 제거해야 합니다.

이때 어떤 블록을 먼저 제거할지 결정하는 기준이 `LRU(Least Recently Used)` 알고리즘입니다.

`LRU`는 **가장 오랫동안 사용되지 않은 데이터 블록을 우선적으로 제거**하는 방식입니다.  
즉, 최근에 다시 사용될 가능성이 낮은 블록부터 내보내고, 최근에 자주 접근된 블록은 더 오래 메모리에 유지합니다.

## 단계별 동작 메커니즘

### 1. 새로운 블록 적재

디스크에서 새로운 블록을 메모리에 적재하면, 해당 블록은 가장 최근에 사용된 상태를 의미하는 `MRU(Most Recently Used)` 위치에 배치됩니다.

### 2. 기존 블록의 우선순위 하락

새로운 데이터 블록들이 계속 적재되면, 기존에 있던 블록들은 상대적으로 뒤로 밀리게 됩니다.  
이 과정에서 오래 사용되지 않은 블록은 점차 `LRU` 위치로 이동합니다.

### 3. 재접근 시 위치 갱신

이미 `Cache`에 존재하는 블록에 다시 접근하면, 해당 블록은 다시 `MRU` 위치로 이동합니다.  
즉, 최근에 다시 사용된 블록은 메모리 체류 시간이 연장됩니다.

### 4. 메모리 부족 시 블록 제거

메모리 임계치에 도달해 빈 공간이 부족해지면, 최종적으로 `LRU` 위치에 있는 블록부터 순차적으로 해제됩니다.  
이렇게 확보된 공간에 새로운 데이터 블록이 적재됩니다.

## 정리

- `Buffer Cache`는 디스크 I/O 지연을 줄이기 위해 자주 사용하는 데이터 블록을 메모리에 보관하는 공간입니다.
- 데이터가 메모리에 있으면 `Cache Hit`, 없으면 `Cache Miss`가 발생합니다.
- `LRU` 알고리즘은 가장 오래 사용되지 않은 블록부터 제거해 제한된 메모리 공간을 효율적으로 관리합니다.
- 최근에 다시 접근한 블록은 `MRU`로 이동하고, 오래 접근하지 않은 블록은 `LRU`로 밀려 제거 대상이 됩니다.

결국 `Buffer Cache`와 `LRU`는 데이터베이스가 제한된 메모리 안에서 더 빠르게 동작하도록 만드는 핵심 구조입니다.
